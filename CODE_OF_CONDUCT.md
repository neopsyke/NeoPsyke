# Code of Conduct

## Purpose

NeoPsyke aims to be a technically serious, respectful project. Collaboration
should be direct, honest, and rigorous without becoming hostile or dismissive.

This code of conduct applies to repository discussions, issues, pull requests,
code review, and other project spaces.

## Expected Behavior

- Be respectful and professional.
- Critique ideas, code, and design decisions, not people.
- Assume good faith, but be explicit about technical concerns.
- Prefer evidence, examples, and reproducible reasoning over rhetoric.
- Accept correction when shown a mistake.
- Keep discussions focused on moving the work forward.

## Unacceptable Behavior

- Harassment, threats, intimidation, or personal attacks
- Discriminatory or degrading language
- Repeated bad-faith argumentation or deliberate misrepresentation
- Trolling, pile-ons, or hostile review behavior
- Publishing private or sensitive information without permission
- Using project spaces for spam or unrelated promotion

## AI-Assisted Contributions

AI-assisted contributions are welcome, but contributors remain responsible for
the correctness, licensing, security, and maintainability of what they submit.

The same behavioral standards apply regardless of whether a contribution was
written by a human, an agent, or a hybrid workflow.

## Enforcement

Project maintainers may edit, hide, lock, or remove content that violates this
code of conduct. Maintainers may also reject contributions or restrict access
for repeated or serious violations.

## Reporting

To report a conduct issue, contact:

- `hello@neopsyke.ai`

Reports will be handled as privately as practical.

## Scope

This code of conduct applies within project spaces and when someone is
representing the project in public collaboration contexts.
